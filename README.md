# 🌐 现代 Web 开发参考手册（AI 辅助版）

> 面向：Web 初学者、信息与计算科学/信息资源管理相关专业学生、想快速搭建科研工具的人。

本仓库是一份我在毕设过程中整理的系统性 Web 开发知识手册，涵盖从基础概念到项目落地的完整链路。它不是传统意义上逐条罗列语法的“教程”，而是一张以“理解系统运行逻辑”为目标的开发地图——帮助你搞懂 Web 系统全貌，学会用 AI 辅助开发，遇到报错知道去哪查。

## 📖 手册简介

这本手册的写作出发点是：传统 Web 学习通常从“HTML 标签 → CSS 属性 → JavaScript 语法 → 服务器配置 → 数据库连接”开始，很多人学完后知道“这个标签怎么写”，但仍然不清楚“一个网站到底是怎么跑起来的”。

现在做 Web 开发，早就不用一个人硬啃几万行代码。这本手册的核心逻辑是：
你提出需求 → AI 生成代码 → 你看懂结构 → 你修改维护 → 系统上线

text

因此手册不讲具体语法细节，而是聚焦于三个任务：
1. **搞清楚概念**（这玩意儿是干嘛的）
2. **看懂系统架构**（它放在哪一层、跟谁打交道）
3. **学会怎么跟 AI 描述需求**（让它帮你干活）


## 🗂️ 手册目录

手册共 12 个章节，从 Web 基础知识覆盖到项目实战落地：

- **第 1-3 章**：Web 本质、服务器、浏览器、三大核心板块（前端/后端/数据库）
- **第 4 章**：前端基础（HTML/CSS/JavaScript）
- **第 5 章**：现代前端框架概述（Vue/React/Angular）
- **第 6 章**：后端基础（服务器、API、动态网页技术演进）
- **第 7 章**：数据库（关系型数据库、SQL、ORM）
- **第 8 章**：HTTP 通信（请求方法、状态码、JSON、Ajax）
- **第 9 章**：用户登录与权限（登录流程、Token/JWT）
- **第 10 章**：项目开发完整流程（需求分析→系统设计→技术选型→开发→联调→部署）—— **已同步至 v2.0 实际开发状态**
- **第 11 章**：AI 配合模板（前端/后端/数据库/报错/部署/RAG 等 12 类模板，**新增 3 个 v2.0 场景模板**）
- **第 12 章**：报错排查速查表（前端/后端/联调/Flask 专用）

手册采用口语化风格，以“南农教务系统”“农业文献检索系统”等贴近信息资源管理专业的场景贯穿始终，便于理解和迁移。


## 🔗 配套项目
